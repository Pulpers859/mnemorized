GEMINI IMAGE GENERATION CONSTITUTION
=====================================
Single source of truth for all Mnemorized prompt generation and repair.
Both stress_visual_pipeline.py and visual_qa_pack.py load this file at
runtime. Edit HERE — never maintain inline copies in Python code.

Last updated after Forge-first stress-test correction (19 failure modes,
visual encoding hierarchy + metaphor library + no-grid plate guard).

═══════════════════════════════════════════════════════════════════════════
WHAT GEMINI CAN RENDER RELIABLY
═══════════════════════════════════════════════════════════════════════════

- Distinct objects with clear silhouettes (characters, animals, props, furniture)
- Specific materials and states (glass, metal, liquid, fire, broken, deflated, glowing)
- Text labels of 1-4 words on objects (plaques, stamps, signs, gauge faces)
- Coarse spatial placement: left, center, right, foreground, background
- Size contrast between objects (small vs massive, close vs distant)
- Artistic styles (ink-and-watercolor, hand-drawn, cartoon) and lighting (warm amber, spotlight)
- Color and palette control (warm tones, monochrome, vivid vs muted)

═══════════════════════════════════════════════════════════════════════════
WHAT GEMINI CANNOT RENDER — 19 confirmed failure modes
═══════════════════════════════════════════════════════════════════════════

1. MICRO-POSES: interlocked fingers, balloon-puffed cheeks, specific hand/joint positions.
   Silently ignored; extra text dilutes instructions Gemini CAN follow.
   FIX: Remove the micro-pose. Let the character's prop or action carry the meaning.

2. GROUND/SURFACE CONTRAST: "pristine sawdust here, scorched ground there."
   Gemini applies effects to the nearest cause, not where you specify.
   FIX: Drop surface descriptions. Let contrast live in objects, not floors.

3. COMPOUND SPATIAL POSITIONS: "center-left" renders as center.
   FIX: Use single-axis terms only (left, center, right, foreground, background).

4. SHAPE-MORPHING: flames shaped like letters, clouds forming words.
   FIX: Use separate labeled objects. "A plaque reading X" not "flames spelling X."

5. TOO MANY SPATIAL CONSTRAINTS: more than ~3 per figure.
   Gemini honors 1-2 and ignores the rest.
   FIX: Describe each figure with at most 2 spatial constraints.

6. CLOCK HAND POSITIONS: "hands frozen at one o'clock" is silently ignored.
   FIX: Use text plaques ("1 HR") instead of clock hand angles.

7. AMBIGUOUS MOTION VERBS: "crossing the line" may render as approaching.
   FIX: Use completed-state language: "has crossed the line," "boot planted past the line."

8. ACTION-STATE AMBIGUITY: "ready to exhale" or "about to X" renders as actively doing X.
   FIX: For idle/waiting, describe completed inactivity: "arms folded," "standing still."

9. ALL-CAPS TEXT LEAKAGE: any ALL-CAPS word in the prompt may render as visible text
   in the image, even scene directions (FOREGROUND, ALREADY CROSSED, LEFT RING).
   FIX: Reserve ALL-CAPS exclusively for labels that MUST appear in the final image.
   Use lowercase for all emphasis, directions, and instructional language.

10. DESCRIPTIVE-NOUN TEXT LEAKAGE: naming the TYPE of label in the prompt
    ("pharmacy label", "warning sign", "product tag") causes Gemini to render the
    descriptor noun as visible text ("PHARMAC", "WARNING", "PRODUCT").
    FIX: Describe visual properties only. "white sticker" not "pharmacy label."
    "red sign" not "warning sign." Never name what kind of label it is.

11. SCREEN-TEXT CHARACTER DUPLICATION: glowing/digital screen text duplicates
    adjacent identical characters. "cEEG" (2 E's) renders as "cEEEG" (3 E's).
    FIX: Use physical-text devices (etched, engraved, stamped) for any label
    containing adjacent repeated characters. Reserve glowing screen text for
    labels with no adjacent identical characters.

12. MULTI-WORD LABEL DUPLICATION: labels of 2+ words painted on large surfaces
    (walls, floors) cause Gemini to split and duplicate the first word.
    "RV STRAIN" renders as "RV" + "RV STRAIN" (word duplication).
    FIX: Shorten multi-word labels by letting the visual context carry part
    of the meaning. Use a 1-word label on a precision device (gauge, plaque)
    and let the scene element itself convey the second concept.
    Example: "RV" engraved on a gauge + bulging cracked wall = "RV strain."

13. CHARACTER ANATOMY OVERFLOW: requesting more held items than a character
    has hands (e.g., 2 swords + 1 shield = 3 items for 2 arms) causes Gemini
    to render floating or anatomically impossible objects. Items appear with
    no hand holding them, producing obvious "AI slop."
    FIX: Count held items per character. Maximum 2 held items (one per hand).
    If a character needs to carry more, attach extras to their body (belt,
    back strap, shoulder sling) or place them on a nearby surface.
    Pre-submit checklist: for every character, verify held_items <= 2.

14. OBJECT DUPLICATION: when a single instance of a unique prop is intended
    but the prompt does not constrain quantity, Gemini may render 2-3 copies
    to fill visual space (e.g., one B6 bottle becomes two).
    FIX: For unique props, state "one single [object]" and add "no other
    [objects] anywhere in the scene." Only needed for small standalone props;
    architectural elements (pillars, gears) can repeat naturally.

15. META-INSTRUCTION CAPTION LEAKAGE: words such as "encode," "anchor,"
    "station," "represents," "clearly reads as," and topic-role names can
    become visible caption strips or booth labels. This turns mnemonic scenes
    into labeled infographics.
    FIX: In the final image prompt, describe only the object and its visible
    state. Do not say what it encodes inside the generation prompt. Add an
    explicit ban on captions, subtitles, booth titles, explanatory text, role
    labels, numbered medallions, and black text strips.

16. SEGMENTED-GRID OVERFITTING: forcing many anchors into separate booths,
    bays, numbered panels, or tiled cells produces an infographic/checklist,
    not a memory palace. This is especially tempting after a dense-scene
    failure, but it destroys the one-room spatial map that Mnemorized needs.
    FIX: Plates are focused image attempts over 1-3 anchors inside the same
    coherent room, not grid cells. Use left/center/right/foreground/background
    zones, object relationships, and a reading path through one scene. Use
    explicit booths/panels only as a rare last resort when the source metaphor
    itself demands it and the user accepts the tradeoff. Default to no grids.

17. PACKAGING/STICKER TEXT HALLUCINATION: boxes, cartons, commercial-looking
    containers, stickers, and dial faces invite Gemini to invent small nonsense
    words even when text is restricted.
    FIX: If text is not essential, use an unmarked physical object instead:
    clay bowl, plain vial, unlabeled sack, naked device, colored liquid, or
    silhouette-only prop. If a label is essential, put one exact label on a
    large physical device and include it in the allowlist.

18. SLEEP-MARK TEXT LEAKAGE: words like "sleeping," "asleep," "sleepy," and
    "zzz" can produce visible letter-like sleep marks. This is a text leak
    when the image requires no text.
    FIX: Show inactivity mechanically: lens cap on, unplugged cord, switch off,
    covered with cloth, arms folded, gate locked, object resting on the ground.
    Avoid sleep-language in strict no-text prompts.

19. MIXED-CASE CHEMICAL LABEL FRAGILITY: labels where case changes meaning,
    especially chemical abbreviations like CaCl, can render as CaCI, CACL, or
    all-caps variants. The learner usually understands the cue, but exact
    letterform is not reliable enough for catalog-grade precision.
    FIX: Prefer full-word precision labels such as "chloride 1g" when space
    allows, or use a non-text chloride cue attached to the calcium object. If
    the abbreviation must appear, audit it manually and do not call the image a
    clean pass if the letterform is ambiguous.

PROVEN POSITIVE FORMULAS — what consistently produces A+ (>=96) images
═══════════════════════════════════════════════════════════════════════════

VISUAL ENCODING HIERARCHY:
Before assigning a text label to an anchor, check this priority order:

  1. PURE VISUAL METAPHOR — no text needed.
     Use when the object has an instant, universal association.
     Examples: banana barrel = potassium, turtle = go slow, skull = death.
     See docs/visual-metaphor-library.txt for the full catalog.

  2. HYBRID — metaphor object with a small integrated label.
     Use when the metaphor carries most of the meaning but a short label
     disambiguates. The label goes ON the metaphor object, not a separate
     plaque.
     Examples: giant salt shaker with "NS" etched on cap, syringe with
     "subQ" engraved on barrel.

  3. TEXT LABEL ONLY — use the mixed-device vocabulary below.
     Use only for specific numeric thresholds, doses, and acronyms that
     have no visual equivalent.
     Examples: "0.1 U/kg/hr", "pH <6.9", "AG", "CTPA", "PESI".

This hierarchy reduces text labels per image (improving Gemini reliability)
and produces more memorable visual mnemonic illustrations without referencing
or copying commercial mnemonic products.

LABEL READABILITY — MIXED-DEVICE VOCABULARY:
Integrate text into objects naturally using the device that fits the object type.
Each device has been stress-tested for character precision. Listed by reliability:

  1. Engraved/etched metal — shields, bezels, frames. HIGHEST precision.
     "The word [LABEL] is deeply engraved into the face of his metal shield."
  2. Chiseled stone — blocks, walls, foundations. High precision.
     "The stone block has [LABEL] chiseled into its surface in deep block letters."
  3. Riveted/stamped metal — consoles, machinery, industrial. High precision.
     "The console has [LABEL] stamped in raised riveted metal letters."
  4. Painted on surface — jars, dials, clock faces. Good precision.
     "Painted directly on the glass in bold dark letters: [LABEL]."
  5. White sticker/printed — bottles, containers. Good precision.
     "Printed on a small white sticker in bold dark text: [LABEL]."
     NEVER name the label type (see failure mode #10).
  6. Glowing screen text — monitors, displays. Avoid repeated characters (#11).
     "The screen displays [LABEL] in glowing green digital text."

FALLBACK — BRASS PLAQUE FORMULA (legacy, still reliable):
For cases where no natural device fits, or as a safe default:
  "A large brass plaque bolted to [object] reads [LABEL] in bold block letters."
Produces readable text but looks uniform — prefer mixed devices when possible.

TEXT ALLOWLIST FENCE:
Always end the prompt with:
  "Visible text limited to: [comma-separated list]. No other text, no floating
  captions, no zone labels, no speech bubbles, no directional words anywhere
  in the image."
This prevents spurious text from leaking into the scene.

For any scene, strengthen the fence when text leakage appears:
  "No captions, no subtitles, no explanatory text, no numbered medallions, no
  speech bubbles, no floating labels, no black text strips."

STYLE OPENER:
Begin every prompt with:
  "Detailed hand-drawn medical mnemonic illustration in warm ink-and-watercolor
  style, wide 16:9 format, every element large and silhouette-readable."

READING PATH:
Include a reading path in the second sentence:
  "The eye enters at [entrance], sweeps through [zones], climbs to [back], and returns."

COMPLETED-STATE LANGUAGE:
For sequence/timing anchors, describe the completed state:
  "has crossed the line" not "crossing"
  "has swung open" not "opening"
  "boot planted past the line" not "stepping over the line"

MULTI-PART VERIFICATION:
When a feature requires an exact count (e.g., three-headed dog), add:
  "All three heads are clearly visible and separated."

SHAPE-ENCODING WORKAROUND:
If you need "object A represents concept B through shape," use a recognizable
version of B as a SEPARATE object attached to or replacing A.
  Example: "one stool leg is REPLACED BY a dangling deflated blood pressure cuff."

PLATE WORKFLOW:
A plate is a focused render of a subset of anchors inside the same memory-palace
room. It is not a tiled grid, bay layout, storyboard, or infographic panel.
For plate prompts, reuse the established room and place 1-3 large anchor beats
in coherent zones. The final assembled set can contain multiple plates, but each
plate should still feel like a real scene/crop from the palace.

GRID/BAY EXCEPTION STANDARD:
Segmented grids, separate bays, storyboard panels, and tiled booth layouts should
almost never be used. They look cheap, flatten the mnemonic into a checklist,
and should be considered a prompt-design failure unless there is a specific,
approved reason to use that metaphor.

PROCESS RULES
═══════════════════════════════════════════════════════════════════════════

FORGE-FIRST STRESS TESTING:
Do not treat hand-authored mini-prompts as Mnemorized app validation.
For serious medical topics, start from the actual Forge workflow:
  1. Run the topic in Forge as a user would.
  2. Run/inspect the Medical Quality Gate when private retrieval is available.
  3. Confirm citations are relevant to the topic and not merely nearest-neighbor
     noise from unrelated ingested material.
  4. Export the Forge bundle.
  5. Build the QA pack from the bundle with tools/visual_qa_pack.py.
  6. Send the saved full-scene or plate prompt to Gemini/Antigravity.
  7. Audit the image against the Forge anchor table, narration, and encodes.

Synthetic prompts are only renderer probes. They may test whether Gemini can
render one prop, one label, or one composition idea. They cannot prove medical
coverage, source grounding, board relevance, or catalog readiness. If a prompt
troubleshooting run did not start from a Forge bundle, label it explicitly as a
renderer experiment, not a Forge stress test.

SOURCE-GROUNDING GATE:
For medical board topics, do not silently replace missing source retrieval with
generic medical knowledge. If citations are absent, unrelated, or drawn from the
wrong textbook chapter/topic, report the retrieval failure and fix ingestion or
querying before judging the generation as source-backed.

PRE-IMAGE STRUCTURAL GATE:
Do not send a Forge prompt to Gemini/Antigravity just because the story exists.
If the auto structural audit is below 85, or if the story contains severe visual
text overload, repair/retry the Forge story first. Known blockers: visuals over
30 words, more than two text-bearing surfaces in one anchor, arrow glyphs,
speech bubbles, checklist rows, and label lists. A 71/100 pre-image prompt
predictably produces a weak image; a cleaned 85+ prompt is the minimum worth
spending image-generation effort on.

FULL-PROMPT REGENERATION OVER REPAIR PATCHES:
When fixing issues, rewrite the COMPLETE prompt with fixes embedded.
Targeted repair instructions cause previously-working elements to be dropped
during regeneration. A full prompt preserves composition.

EXTERNAL AUDITOR — NEVER SELF-GRADE:
The image generator (AG/Gemini) must NOT audit its own output.
Self-grading inflates scores by 15-20 points.
Claude audits externally against the anchor table.

PROMPT LENGTH TARGETS:
- Single-anchor plates: 150-250 words
- Full-scene prompts (all anchors): 400-600 words
- If over limit, cut room/setting description first, then secondary visual details.
  Never cut the core mnemonic hook or required labels.

REPAIR LOOP LIMITS:
- Repairs must be SHORTER or equal to the original prompt.
- Never add micro-poses in repairs.
- Rewrite the ENTIRE prompt on repair — do not patch individual sentences.
- Maximum 5 repair iterations per plate.
- After 5 failures, redesign the anchor's hook, not the prompt wording.

TEXT BUDGET:
- Target 4-6 text labels per image after applying the visual encoding hierarchy.
- Pure metaphors and hybrid encodings reduce the text count, improving reliability.
- Each label: 1-4 words maximum.
- Budget text carefully — every label competes for Gemini's attention.

PRE-SUBMIT CHECKLIST (run before sending any prompt to AG):
- [ ] Every character holds <= 2 items (one per hand). No 3-item combos.
- [ ] Unique props explicitly constrained: "one single [X], no other [X]."
- [ ] No ALL-CAPS words except intended visible labels.
- [ ] No descriptor nouns naming label types ("pharmacy label", "warning sign").
- [ ] Labels with adjacent repeated characters use physical devices, not screens.
- [ ] Multi-word labels on precision devices, not painted on large surfaces.
- [ ] Visual encoding hierarchy applied: metaphor first, text last resort.
- [ ] No meta-instruction words in the final image prompt: encode, anchor,
      station, represents, role label, or "clearly reads as."
- [ ] No segmented grids, booths, bays, storyboard panels, or checklist cells
      by default. If one appears, treat it as a likely failure.
- [ ] Text allowlist fence present at end of prompt.

SESSION COST PROTOCOL — mandatory for all stress-test and iteration loops
═══════════════════════════════════════════════════════════════════════════

REMINDER: /compact after every AG round-trip or commit; /clear between topics.

DISPATCHER MODEL:
Opus (main conversation) decides and reviews only. It does NOT write prompt
files, audit reports, or task files inline. All bulk output (400+ words) is
delegated to Sonnet subagents. Main context target: under 50k tokens always.

SONNET DELEGATION:
- Prompt file writing (400-600 words) → Sonnet subagent
- Audit reports (anchor tables, detailed scores) → Sonnet subagent
- Task file writing → Sonnet subagent
- Constitution/library updates (small, targeted edits) → Opus inline OK
- Opus receives 1-2 sentence pass/fail summaries, not full artifacts.

CONTEXT HYGIENE:
- /compact after every AG round-trip (prompt → generate → audit cycle)
- /compact after every git commit
- /clear between topics (when topic A passes, /clear before starting topic B)
- Re-read constitution sections with offset+limit, not full 250+ line reads
- After /clear, re-read only the constitution sections needed for the next task

MCP TOOL DISCIPLINE:
- Chrome/browser MCP tools: /compact immediately after browser work completes
- Only load MCP tools when actively needed for the current step
- Tool results rot in context — flush them with /compact once the step is done

BUDGET GUARDRAILS:
- If session usage exceeds 50%, pause and ask user whether to continue or /clear
- Single session should not consume more than 60% of daily allocation
- If iterating on 3+ topics, each topic gets its own /clear boundary
- Never carry iteration history from a completed topic into the next one

FIELD FINDINGS — 2026-07-07 (DKA metaphor-hooks live run, commit 63105d1)
═══════════════════════════════════════════════════════════════════════════

Context: full live E2E run on a DKA lesson, images rendered via Antigravity.
The pre-fix build was slapping TEXT LABELS on generic barrels/walls instead of
using sanctioned pure-visual metaphors. Root cause and fixes below; record
structural lessons here so future runs inherit them.

POSITIVE — structural components that worked, KEEP and reinforce:

1. INJECT THE SANCTIONED LIBRARY INTO THE LIVE PROMPT.
   The static frontend cannot read docs/ at runtime, so the metaphor catalog
   was never reaching Stage-2 generation. Embedding it as VISUAL_METAPHOR_LIBRARY
   in forge-pipeline.js and injecting it into the story prompt is what actually
   changed model behavior. A hierarchy the model never sees does nothing —
   the library MUST be inlined in the prompt, not just documented here.

2. "METAPHOR BEATS THEME" is essential (bizarreness effect).
   Adding tier-1 PURE VISUAL METAPHOR alone was not enough — the model still
   swapped a sanctioned metaphor for a theme-matching labeled prop to keep the
   scene coherent. An explicit rule that a sanctioned metaphor ALWAYS wins over
   scene-theme coherence (an incongruous, oddly-placed iconic object is MORE
   memorable than a theme-consistent labeled barrel) is what made the metaphors
   actually render. Do NOT let environment coherence suppress an off-theme hook.

3. THE HIERARCHY CORRECTLY KEEPS NON-METAPHOR TIERS FOR DECISIONS.
   Not every anchor should be forced into a bare metaphor. A threshold DECISION
   (K+ replacement bands) rendered best as a functional/contrast hook
   (traffic-light paddles), not a forced banana. Tier 1 is for instant universal
   associations; thresholds/doses legitimately live lower in the hierarchy.
   Validated live: salt shaker = sodium, sugar sack = glucose, skull = cerebral
   edema, hourglass = monitoring interval, pouring ladle = insulin delivery
   (NOT "barrel labelled INSULIN"), traffic-light paddles = K+ bands.

NEGATIVE — failure modes and over-broad rules to adjust or watch:

1. WEAK HOOKS ORIGINATE UPSTREAM, AT STAGE-2 STORY, NOT AT THE IMAGE PROMPT.
   The visual-field choice (what object represents each fact) is made when the
   story/anchor script is generated. Fixing only the image-prompt builder cannot
   recover a hook the story already chose as "a barrel with a label." Any future
   hook-quality fix must land in the Stage-2 story prompt first.

2. AN OVER-BROAD ORIGINALITY RULE SUPPRESSES SANCTIONED METAPHORS.
   The prior "every scene and symbol must be original / do not reuse known
   commercial mnemonic symbols" wording caused the model to avoid banana=K+,
   salt=NaCl, baking-soda=bicarb and collapse to text labels. The IP boundary
   forbids only PROPRIETARY curated systems (Sketchy/Pixorize scenes, character
   names, layouts, fact->symbol maps) — NOT common-knowledge associations.
   Keep originality scoped to scenes/characters/layouts; never to the sanctioned
   universal vocabulary.

3. ANCHOR DENSITY CEILING ~ 8-10 PER PLATE.
   At 10 anchors AG rendered ~5 cleanly on the first pass with right-side
   crowding. Dense plates degrade both render fidelity and audit reliability.
   Prefer <=8 encoded anchors per illustration; split lessons rather than
   overload one scene.

4. THE IN-APP VISION AUDIT UNDER-DETECTS ON DENSE PLATES.
   Scores plateaued ~52 and the auditor called present, clearly-labeled anchors
   "entirely absent." Treat sub-target scores on crowded plates as possible
   auditor miss, not proof of a bad image — corroborate visually before
   iterating. Do not burn AG round-trips chasing a hallucinated deficiency.

5. PRECISION TEXT IS ALLOWED BUT CLUSTERS — KEEP IT SPARSE.
   Formulas/rates on a single device (anion-gap door, fluid-rate barrel) are
   legitimate per the hierarchy tier 3, but several on one plate re-create the
   text-density problem. Budget precision labels the same as any other text.

6. MEDIA-TYPE TRUST (harness note). AG may save a JPEG under a .png name; the
   in-app audit trusts the blob type inferred from extension. Real app data-URL
   path is unaffected, but audit harness must match declared media type to bytes.
